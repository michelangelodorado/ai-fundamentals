1 The Attack Surface

One picture to hang the whole section on.

Threats enter through inputs (what the user types and what the model retrieves) and through outputs (what it sends back). Guardrails are the enforcement layer on both sides — the part an attacker can't talk their way past.

Live: the request flow & where guardrails sit

Input guardrails catch injection/PII/secrets before the model; output guardrails catch leakage/toxicity/secrets before the user.

2 Direct vs Indirect Injection ★ #1 risk

The root cause never goes away: to the model, instructions and data are the same tokens.

Direct injection is typed at the model. Indirect injection is far scarier: the payload is planted in content the model later reads — a webpage, an email, a RAG document — and fires when ingested. The victim never sees it.

3 Jailbreaks

A cousin of injection — but aimed at a different target.

Jailbreaks bypass the model's safety training to get content it should refuse (role-play, hypotheticals, encoding, "DAN" personas). Keep the distinction sharp:

Injection vs jailbreak — what's the target?
🎯 Prompt injection
targets your app's instructions — "ignore your system prompt and…"
🎯 Jailbreak
targets the model's safety training — "pretend you have no rules…"

4 Data Leakage & Model Poisoning

What the attacker gets out — and what they corrupt going in.

Data leakage: the model reveals another tenant's data, PII, secrets, or its own system prompt. In RAG, this is usually a missing permission filter (Section 6). Model poisoning: corrupting the training/fine-tuning data to plant a backdoor or bias — attacking the model at its source.

5 Supply Chain Attacks

You downloaded a 70B model — do you actually trust it?

The AI supply chain is huge and mostly pulled from the internet: model weights, datasets, embedding models, packages, container images. A tampered artifact anywhere compromises you — the same dependency problem you know from software, now with model weights.

Live: inspect the supply chain

6 Tool & Agent Abuse ★ the frontier

When the model can act, injection stops being about words and starts being about actions.

Give an agent tools + autonomy, then hijack it with an indirect injection, and it will pursue the attacker's goal across multiple steps — using its own legitimate tools. A single poisoned document becomes an insider threat.

Live: watch the hijack chain unfold
Highest severity: injection + tools + autonomy. This is why guarding agent inputs (and constraining tool access) is the frontier of AI security.

7 Guardrail Scanners

The hard external layer — inspect every input and output.

Prompt rules are soft (the model can be talked out of them). Guardrails are a separate enforcement layer that scans traffic independently: injection, PII, secrets, toxicity. Watch a risky prompt run the gauntlet.

8 PII & Secrets Redaction

    
Catch personal data and credentials in both directions.

    
PII detection redacts names, emails, IDs, card numbers (GDPR / APAC PDPA). Secrets detection catches API keys and passwords developers paste in constantly — before they reach an external model or leak back out.

9 Output Filtering

    
The last line of defense — scan the response before the user sees it.

    
Even a well-guarded input can produce a bad output. Output filtering catches leaked secrets, system-prompt disclosure, toxic or off-topic content on the way out.
