1 Pillars + the AI Layer

Observability = answering "what's happening inside, and why?" from the outside.

The classic three pillars — traces, metrics, logs — still hold. AI just stacks an extra layer on top: tokens, TTFT, cost, quality. Same reason an AI gateway differs from a normal one — the workload is different, so the telemetry must be too.

2 OpenTelemetry (OTel)

Instrument once, send anywhere.

OpenTelemetry is the vendor-neutral open standard for emitting traces, metrics, and logs — so you're not locked into one backend. There's now a GenAI semantic convention that standardizes how LLM attributes (model, tokens) are named. It's the plumbing everything else rides on.

3 Tracing

Follow one request end-to-end.

A trace breaks a single request into timed spans across every component. For a RAG request, the waterfall shows exactly where the time went — and where a failure happened.

Live: a RAG request's span waterfall

4 Metrics

Aggregated numbers for the whole fleet.

Traces tell you about one request; metrics tell you about all of them — request rate, error rate, p95 latency — cheap to store and perfect for dashboards and alerts.

Live: a mini metrics dashboard

5 Token Usage

The fundamental unit — everything else is built on it.

Every request has an input and output token count. Tokens drive cost, latency, and rate limits, so you track them per request, per user, per model.

Live: tokens accumulate per request

6 TTFT vs TPS ★ the key idea

Because responses stream, "latency" is two different numbers.

TTFT (Time To First Token) = how long before it starts talking — perceived responsiveness, the UX metric. TPS (Tokens Per Second) = how fast it talks after that — throughput, the speed metric. A model can be great at one and poor at the other.

Live: compare two models streaming the same answer
Model A — slow TTFT, fast TPS
Model B — fast TTFT, slow TPS
Take this to a call: knowing "latency" = TTFT (UX) + TPS (throughput) is the metric that shows you understand LLM serving, not just web serving.

7 Cost Monitoring

FinOps for AI — tokens × price = money.

Spend varies wildly by model and request, so you track it per user, team, feature, and model — for chargeback, budget alerts, and catching a runaway agent burning tokens.

Live: watch spend accrue against a budget

8 Prompt Logs & Privacy

The data you need to debug is the data you must protect.

Logging prompts and responses is essential for debugging, audit, and building eval sets. But those logs contain PII and secrets (Section 7) — so observability and security collide: logs must be access-controlled and often redacted before storage.

9 Evaluation Metrics

    
The hard part: measuring quality, not just speed.

    
There's no "HTTP 200" for was the answer good? So you measure proxies — faithfulness, relevance, correctness, toxicity, hallucination rate — often using LLM-as-a-judge (one model scoring another's output).
