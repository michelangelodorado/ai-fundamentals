1 Attention — Query, Key, Value

The engine of every modern LLM, made concrete.

Each token creates three vectors: a Query ("what am I looking for?"), a Key ("what do I offer?"), and a Value (its actual content). The model matches one token's Query against every Key to get relevance scores, turns those into weights, and pulls a weighted blend of the Values.

2 Self-Attention & the n² Problem

"Self" means every token attends to every other token — in the same sentence.

That all-pairs comparison is powerful, but the number of connections grows with the square of the length. Double the tokens → 4× the work. This single fact is why long context is expensive and why every trick in this section exists.

3 Positional Encoding

Attention alone is orderless — position has to be added in.

Without position info, "dog bites man" and "man bites dog" look identical to the model. Positional encoding tags each token with where it sits, so order — and therefore meaning — is preserved.

Live: same words, swapped positions
In practice: modern models use RoPE (rotary position embedding), which also helps them stretch to longer inputs than they trained on.

4 KV Cache ★ the big one

The single most important concept for cost and speed.

To generate each new token, the model needs the Keys and Values of every previous token. Recomputing them all every step is hugely wasteful — so we cache them and only compute the new token's. But that cache grows with every token and lives in GPU memory — it's the real bottleneck you size hardware for.

Your world: KV-cache size is what balloons GPU memory in long chats — the practical constraint behind AI infrastructure sizing.

5 Speculative Decoding

Free speed: a tiny model guesses, the big model checks.

Generation is slow because it's one token at a time. A small, fast draft model races ahead and proposes several tokens; the big model then verifies them all in a single pass. Correct guesses are kept for free; wrong ones are thrown out and corrected. Output quality is identical — the big model always has the final say.

6 Chain of Thought

Letting the model "think out loud" makes it dramatically smarter.

Because it's a next-token predictor, writing out intermediate reasoning steps gives the model more compute and scratch space before committing to an answer. The same model, same weights — just allowed to think — gets far more math and logic right.

7 Function / Tool Calling

How an LLM reaches outside its own text.

The model can't actually check the weather — it only writes text. So instead of answering, it emits a structured call. Your app runs the real function and hands the result back, and the model continues with real data. ("Function calling" and "tool calling" mean the same thing.)

Live: the round-trip

8 Agents

An LLM in a loop — thinking, acting, and observing until the goal is met.

A single prompt gives you one answer. An agent runs a cycle: Think → Act (use a tool) → Observe → repeat, deciding its own next step each time. That autonomy is what makes agents powerful — and what makes them a serious security surface, because they take real actions.

Live: the agent loop
Why it matters: agents that take actions autonomously are exactly the surface AI-security controls (input/output guardrails, tool-access policy) are built to govern.
