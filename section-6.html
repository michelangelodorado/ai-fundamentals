1 The Big Idea

RAG is an open-book exam for the LLM.

An LLM only knows its training data — not your private docs, nothing recent — and its context window is too small to paste in everything. RAG retrieves the few relevant snippets from your knowledge base and injects them into the prompt, so the model answers from them.

2 Chunking ★ #1 cause of bad RAG

Documents are too big to embed or retrieve whole — so you split them.

Chunk too big → retrieval drags in lots of irrelevant text. Chunk too small → you lose the surrounding context. The sweet spot (with a little overlap) is where good RAG begins.

3 Embeddings & Vector Space

Each chunk becomes a point; nearby points mean similar things.

Every chunk is embedded into a vector (Section 1). To answer a question, you embed the question too and find the chunks whose points sit closest to it.

4 Semantic vs Keyword Search

Meaning vs exact words — each catches what the other misses.

Keyword search needs shared words. Semantic search matches meaning — so "reset my password" finds "credential recovery process" with zero words in common.

Live: search "how do I reset my password?"

5 Hybrid Search

Best of both — and usually the winner for enterprise data.

Semantic search nails meaning but can fumble exact terms — error codes, product names, IDs. Hybrid runs both semantic and keyword search and merges the results. For data full of jargon and IDs, hybrid almost always beats pure vector.

6 Metadata Filtering ★ where security lives

Constrain retrieval before it happens.

Each chunk carries metadata — source, date, department, access level. Filtering by it means "only retrieve docs this user is allowed to see." That permission filter is how RAG stops the model leaking data a user shouldn't get.

Live: run the query as different users
The security story: permission-aware metadata filtering (don't retrieve what the user can't see) is the front line of RAG data protection — and a natural policy point for a gateway.

7 Retrieval & Reranking

Fetch broad and fast, then re-score for precision.

First retrieval is fast but rough. A reranker — a second, more accurate model — re-scores the top candidates and reorders them. Classic pattern: retrieve top-50 fast, rerank to the best 5. Big quality gain for little cost.

Live: retrieve, then rerank

8 Context Injection

The "augmented generation" — where retrieval meets the prompt.

The top reranked chunks are dropped into the prompt alongside the question, usually with "Answer using only the context below." Then the LLM generates a grounded answer.
